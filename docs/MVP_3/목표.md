# Trading Exchange Engine - MVP Phase 3 목표

## 0. 범위 고정

> Phase 3는 엔진 로직 고도화 + 에러 처리 표준화 + 도메인 이벤트 발행까지만 다룬다.

- [x] 주문 규칙 확장: `TIF(GTC/IOC/FOK)`
- [x] MARKET BUY `quantity/quoteQty` 선택 입력(XOR) 기반 매칭
- [ ] 큐 포화 시 `ENGINE_BACKPRESSURE(503)` 표준 응답
- [ ] 도메인 이벤트(in-process) 발행 지점 표준화
- [x] B-1/B-2/B-3 선행 버그 수정

### 제외 범위

- [ ] 잔고/홀드/정산
- [ ] DB 영속성 완료

## 2. 도메인 확장

### 2.1 TimeInForce

- [x] `TimeInForce { GTC, IOC, FOK }` 추가
- [x] `Order.tif` 필드 추가 (기본값 `GTC`. MARKET 주문은 `tif`를 사용하지 않으며 명시 시 400 거부)
- [x] MARKET 요청에 `tif` 명시 시 `PlaceOrderValidator` 400(`INVALID_REQUEST`) 반환

### 2.2 MARKET BUY 입력 필드

- [x] `Order.quoteQty` 필드 추가 (MARKET BUY 예산 모드 전용)
- [x] MARKET BUY는 `quantity`/`quoteQty` 중 정확히 하나 필수(XOR)
- [x] MARKET BUY 입력 검증
  - 둘 다 누락/동시 입력 시 `PlaceOrderValidator`에서 400(`INVALID_REQUEST`) 처리

## 3. 매칭 규칙 구현 (Phase 3-1/3-2)

### 3.1 IOC

- [x] 즉시 체결 가능한 수량만 체결
- [x] 잔량은 오더북 적재 없이 즉시 `CANCELLED`

### 3.2 FOK

- [x] `OrderBook.totalAvailableQty()` 구현
- [x] 사전 충족성 검사 통과 시에만 매칭 수행
- [x] 미충족이면 체결 없이 `CANCELLED`
- [x] 구현 메모: Phase 3은 선형 스캔, 성능 최적화는 Phase 4 이관

### 3.3 MARKET BUY (`quantity`/`quoteQty`)

- [x] `quoteQty` 모드
  - `remainingQuote` 기준으로 체결 수량 계산
  - 루프 내 체결 건수(`executedTradeCount`) 집계
  - 루프 종료 후 상태 확정
    - `executedTradeCount > 0` -> `markFilledByMarketBuy()`
    - `executedTradeCount = 0` -> `cancel()`
- [x] `quantity` 모드
  - 기존 MARKET 주문과 동일하게 `remaining` 기반으로 동작
- [x] `remaining` 해석 분리
  - BUY + `quoteQty` 모드: 항상 `0`
  - BUY + `quantity` 모드: 기존 규칙대로 감소
- [x] `quoteQty` 모드 미사용 예산 계산
  - `leftoverQuoteQty = requestedQuoteQty - cumQuoteQty`

## 6. API/스펙 정리

- [x] LIMIT 요청: `tif` 생략 시 `GTC`
- [x] MARKET 요청: `tif` 명시 시 400 거부
- [x] MARKET BUY 요청 XOR 규칙 반영
  - `quantity`/`quoteQty` 중 정확히 하나 필수
- [x] `POST /orders` 응답 명시

- [x] `GET /orders/{orderId}` 응답 필드 명시 (모드별)
  - `requestedQuoteQty` (quoteQty 모드)
  - `requestedQty` (quantity 모드)
  - `cumQuoteQty`
  - `cumBaseQty`
  - `leftoverQuoteQty` (quoteQty 모드)
  - `status` (quoteQty 모드: 0건 `CANCELLED`, 1건 이상 `FILLED`; quantity 모드: 기존 MARKET 규칙)
