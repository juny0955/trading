## 1. 목적

모든 클래스가 동일한 구조/주석/메서드 스타일을 따르도록 공통 템플릿을 정의한다.

---

## 2. 핵심 규칙

- 파일 상단 순서: `package` → `import` → 클래스 JavaDoc → `@Getter` 등 애너테이션 → 클래스 선언
- 클래스 JavaDoc에 역할 + 상태 흐름(필요 시 `<pre>`) + 외부 진입점 명시
- 필드는 의미 그룹으로 묶고, 정렬을 맞춰 가독성 유지
- 섹션 구분선 사용:
  - `// -------------------------------------------------------------------------`
  - `// 섹션명`
  - `// -------------------------------------------------------------------------`
- public API에는 JavaDoc 필수
- 내부 로직은 `private` 메서드로 분리하고 이름으로 의도를 드러냄 (`validateXxx`, `requireXxx`)
- 단일 문장 `if`는 중괄호 생략 가능, 블록이 길어지면 중괄호 사용
- 예외 메시지는 구체적으로 작성하고, 에러 코드는 문자열 상수 규칙을 따름
- 외부 진입점은 최소화하고, 세부 생성/보조 메서드는 `private` 우선

## 3. 클래스 골격 템플릿

```java
package dev.junyoung.trading.{domain}.{subdomain};

import java.util.Objects;

/**
 * 클래스 한 줄 설명.
 *
 * <pre>
 * 상태1 → 상태2 → 상태3
 * </pre>
 *
 * 외부 진입점은 {@link #create(...)}이며, 상태 전이는
 * {@link #activate()} / {@link #cancel()}로만 수행한다.
 */
public class TemplateClass {

    private final String id;
    private final String name;
    private String status;

    // -------------------------------------------------------------------------
    // 생성자
    // -------------------------------------------------------------------------

    private TemplateClass(String name) {
        this.id     = generateId();
        this.name   = Objects.requireNonNull(name, "name must not be null");
        this.status = "INIT";
        validateInvariants();
    }

    private void validateInvariants() {
        if (name.isBlank())
            throw new IllegalArgumentException("name must not be blank");
    }

    // -------------------------------------------------------------------------
    // 팩토리 (진입점)
    // -------------------------------------------------------------------------

    /** 객체를 생성한다. */
    public static TemplateClass create(String name) {
        return new TemplateClass(name);
    }

    // -------------------------------------------------------------------------
    // 조회
    // -------------------------------------------------------------------------

    public String getId() {
        return id;
    }

    // -------------------------------------------------------------------------
    // 상태 전이
    // -------------------------------------------------------------------------

    /** INIT 상태일 때만 ACTIVE로 전이한다. */
    public void activate() {
        requireInit();
        this.status = "ACTIVE";
    }

    // -------------------------------------------------------------------------
    // 내부 헬퍼
    // -------------------------------------------------------------------------

    private void requireInit() {
        if (!"INIT".equals(status))
            throw new IllegalStateException("Invalid status: " + status);
    }

    private static String generateId() {
        return java.util.UUID.randomUUID().toString();
    }
}
```

## 5. 적용 범위

- 기본 적용: `domain/model/entity`, `domain/model/value`, `domain/service`
- 동일 도메인 내 application 계층 클래스도 가능하면 같은 섹션 구조를 유지
- 단순 DTO/Record는 과도한 섹션 분리를 생략 가능
