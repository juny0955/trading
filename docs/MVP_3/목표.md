# Trading Exchange Engine - MVP Phase 3 목표

## 0. 범위 고정

> Phase 3는 엔진 로직 고도화 + 에러 처리 표준화 + 도메인 이벤트 발행까지만 다룬다.

- [ ] 주문 규칙 확장: `TIF(GTC/IOC/FOK)`
- [ ] MARKET BUY `quoteQty` 기반 예산 매칭
- [ ] 큐 포화 시 `ENGINE_BACKPRESSURE(503)` 표준 응답
- [ ] 도메인 이벤트(in-process) 발행 지점 표준화
- [ ] B-1/B-2/B-3 선행 버그 수정

### 제외 범위

- [ ] 잔고/홀드/정산
- [ ] DB 영속성 완료
- [ ] Outbox(이벤트 내구성)
- [ ] 메트릭/운영 관측성 고도화

---

## 1. 선행 버그 수정 (Phase 3-0)

- [x] **B-1** `OrderCommandService.placeOrder()` 순서 변경
  - `submit()` 선행
  - 성공 후 `saveInMemory()` 수행
- [x] **B-2** 잘못된 enum 값 / LIMIT `price=null` 입력을 `ValidationException(400)`으로 변환
- [x] **B-3** `DELETE /orders/{orderId}` 응답 코드를 `200` -> `202 Accepted`로 변경

## 2. 도메인 확장

### 2.1 TimeInForce

- [ ] `TimeInForce { GTC, IOC, FOK }` 추가
- [ ] `Order.tif` 필드 추가 (기본값 `GTC`. MARKET 주문은 `tif`를 사용하지 않으며 명시 시 400 거부)
- [ ] MARKET 요청에 `tif` 명시 시 `ORDER_TIF_INVALID(400)` 반환

### 2.2 MARKET BUY 전용 필드

- [ ] `Order.quoteQty` 필드 추가 (MARKET BUY 전용)
- [ ] MARKET BUY는 `quoteQty` 필수, MARKET SELL은 `quantity` 필수

---

## 3. 매칭 규칙 구현 (Phase 3-1/3-2)

### 3.1 IOC

- [ ] 즉시 체결 가능한 수량만 체결
- [ ] 잔량은 오더북 적재 없이 즉시 `CANCELLED`

### 3.2 FOK

- [ ] `OrderBook.totalAvailableQty()` 구현
- [ ] 사전 충족성 검사 통과 시에만 매칭 수행
- [ ] 미충족이면 체결 없이 `CANCELLED`
- [ ] 구현 메모: Phase 3은 선형 스캔, 성능 최적화는 Phase 4 이관

### 3.3 MARKET BUY (`quoteQty`)

- [ ] `remainingQuote` 기준으로 체결 수량 계산
- [ ] 루프 내 체결 건수(`executedTradeCount`) 집계
- [ ] 루프 종료 후 상태 확정
  - `executedTradeCount > 0` -> `markFilledByMarketBuy()`
  - `executedTradeCount = 0` -> `cancel()`
- [ ] `remaining`은 MARKET BUY에서 항상 `0`으로 해석

## 4. 백프레셔 + 멱등키

- [ ] 큐 포화 시 `EngineQueueFullException` 발생
- [ ] `ENGINE_BACKPRESSURE(503)` 에러코드 매핑
- [ ] `PlaceOrderRequest.clientOrderId` 선택 필드 추가
- [ ] Phase 3 멱등 범위: `clientOrderId` 단독 유일키(in-memory)
- [ ] 동일 `clientOrderId` 재시도 시 기존 `orderId` 반환
- [ ] Phase 4에서 `(accountId, clientOrderId)`로 이관 예정 명시

## 5. 도메인 이벤트 발행 (Phase 3-3)

- [ ] `OrderStateChangedEvent`, `TradeExecutedEvent` 정의
- [ ] 도메인 객체에서 이벤트 적재
  - `activate()`, `fill()`, `markFilledByMarketBuy()`, `cancel()` 호출 직후
- [ ] `EngineLoop`에서 적재 이벤트 회수 후 `ApplicationEventPublisher` 발행
- [ ] 도메인/매칭 로직에 Spring 의존성 미침투 원칙 유지
- [ ] 이벤트 유실 허용(Phase 3), Outbox는 Phase 4에서 도입
